Transition from exploratory landing page to durable doctrine repository.

## Near-Term

- refine `v1.2` wording against real repository use
- test the level model against active internal projects
- define a minimal badge policy that avoids ceremony
- create a short template pack for new repositories

## Medium-Term

- publish a cleaner public-facing landing page derived from the markdown source of truth
- add examples of real repositories at each level
- define a compact AI trust map template
- define a standard `FAILURES.md` format

## Long-Term

- use Conscious Code as a transversal operating standard across the portfolio
- make certification rare and evidence-based
- prove the standard through repositories that are clearly stronger because of it
