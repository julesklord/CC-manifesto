# Conscious Code

Conscious Code is an internal development standard for building software with discipline in the age of AI-assisted coding.

It is not anti-AI.
It is anti-abdication.

The core idea is simple:
- use AI as an instrument, not as a substitute for judgment
- prefer sufficiency over complexity theater
- document what matters
- keep failure visible
- retain operational ownership of the systems you build

This repository is not a product.
It is a working doctrine for how projects should be conceived, built, documented, and maintained.

## Why This Exists

AI coding tools can dramatically increase speed.
They can also increase:
- unearned complexity
- invisible dependency
- cognitive offloading
- fake maturity
- software that works today but cannot be explained tomorrow

Conscious Code exists to resist that drift.

It defines a staged methodology:
- `Explore`
- `Personal Tool`
- `Maintained Project`
- `Certified Conscious Code`

Not every project should carry the same burden.
But every project should be built consciously.

## Core Principles

Conscious Code is built around these commitments:

1. Logic dictates. AI executes.
2. Sufficiency is victory.
3. Conscious decisions, deliberate execution.
4. AI use must justify its real cost.
5. Evidence drives iteration.
6. Purpose over exhibition.
7. Cognitive sovereignty is non-negotiable.
8. Failures are primary documentation.

## Intended Use

This standard is best used as:
- an internal doctrine across multiple repositories
- a review lens for deciding what to keep, freeze, or discard
- a way to prevent “AI-generated progress” from replacing maintainable engineering
- a maturity model for personal tools and flagship projects

## What This Is Not

Conscious Code is not:
- a purity ritual
- a public certification authority
- a ban on AI
- a demand for perfection
- a requirement to over-document disposable work

It is a discipline for keeping speed answerable to reality.

## Working Rule

If a principle cannot survive contact with real work, it is philosophy theater.

If a standard makes all work impossible, it is vanity discipline.

Conscious Code exists between those two failures.
